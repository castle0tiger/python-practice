# 개념 사전 (GLOSSARY)

> 학습 중 등장한 핵심 개념을 찾아보기 쉽게 정리한 사전.
> 기준: ①앞으로 계속 만나는 개념 ②이해에 실제로 시간을 쓴 개념 — 둘 다 해당할 때만 추가.
> 찾는 법: `Ctrl + F` 로 단어 검색.

---

## 웹 / 서버

### 서버 (server)
요청을 받으면 일을 하고 결과를 돌려주는 프로그램. 식당의 주방.
내 컴퓨터에서 돌리면 `127.0.0.1`(내부 전용), 배포하면 인터넷 주소가 생김.

### 백엔드 / 프론트엔드
- 프론트엔드 = 화면 (HTML/CSS/JS, 사용자가 보는 것)
- 백엔드 = 서버 (데이터 처리, AI 호출, DB — 사용자 눈에 안 보이는 것)

### 엔드포인트 (endpoint)
서버의 주문 창구 하나. "이 주소로 오면 이 일을 해준다" = **주소 + 함수 한 쌍**.
`@app.get("/scores")` + `get_scores()` = 엔드포인트 1개.
end+point: 요청이 도착하는 종착 지점.

### 라우팅 (routing)
주소와 함수를 연결하는 것. `@app.get("/주소")` 데코레이터가 하는 일.
"이 주소로 온 손님은 이 창구로 안내해라."

### GET / POST (HTTP 메서드)
요청의 종류. GET = 데이터 달라 (조회), POST = 데이터 보낼게 (전송/생성),
DELETE = 지워줘. GET은 브라우저 주소창으로 가능, POST는 도구(fetch, /docs) 필요.

### JSON
서로 다른 프로그램끼리 데이터를 주고받는 표준 형식. 파이썬 딕셔너리와 생김새가 거의 같음.
`{"이름": "캐슬타이거", "점수": 85}` — FastAPI는 함수가 return한 딕셔너리를 자동으로 JSON으로 변환.

### FastAPI vs Flask
둘 다 파이썬 백엔드 프레임워크. 차이는 **누가 화면을 그리냐**:
- Flask: 서버가 HTML 완성본을 만들어서 줌 (Jinja2 템플릿)
- FastAPI: 서버는 JSON 데이터만 줌, 화면은 브라우저(JS)가 그림
FastAPI 방식이면 웹/앱/다른 서버 어디서든 같은 API를 쓸 수 있음 → 현대 표준.

### /docs (자동 문서, Swagger UI)
FastAPI가 내 코드를 읽고 자동으로 만들어주는 **메뉴판 + 시식 코너**.
내 서버가 뭘 할 수 있는지 보여주고(문서), Try it out으로 바로 테스트 가능(도구).
코드에서 뽑아내므로 항상 코드와 일치함.

### fetch()
브라우저 JavaScript가 서버에 HTTP 요청을 보내는 함수.
`/docs`에서 손으로 하던 요청을 코드로 자동화한 것. `await` = 응답 올 때까지 기다려.

### id / getElementById (HTML ↔ JS 연결)
HTML은 부품에 이름표(id)를 달고, JS는 `document.getElementById("이름표")`로 그 부품을 찾는다.
찾은 뒤 `.value`(입력창 글자), `.innerHTML`(상자 속 HTML 내용)을 읽거나 쓴다.
철자 주의: 마지막이 **Id** (대문자 I) — `getElementByld`(엘)로 쓰면 "is not a function" 에러.

### F12 Console (JS의 traceback)
JS 에러는 서버 터미널에 안 나온다. 브라우저 F12 → Console 탭이 JS의 에러 창구.
`at page:16:34` = 16번 줄 34번째 글자에서 터짐. 버튼이 아무 반응 없으면 무조건 F12부터.

### 파이썬 ↔ JS 대응표
| 파이썬 | JS |
|--------|-----|
| `def send():` | `function send() {` |
| `box = 값` | `const box = 값` |
| `==` | `===` |
| 들여쓰기 블록 | `{ }` 블록 |

### 404 vs 422 vs 500 (에러 번역)
- **404 Not Found**: 그 주소에 등록된 함수가 없음 → 주소 오타, 라우트 확인
- **422 Unprocessable Content**: 보낸 데이터가 불량 → 함수 실행 **전에** 차단됨. 검문소 2개:
  ①JSON 문법 붕괴 (`json_invalid`) ②타입 불일치 (`int_parsing` 등, BaseModel이 차단)
- **500 Internal Server Error**: 함수 실행 중 코드가 터짐 → **서버 터미널의 traceback** 확인
브라우저는 "터졌음"만 알려주고, 이유는 터미널(500) 또는 응답의 detail(422)에 있다.

### BaseModel (Pydantic)
받을 데이터의 양식을 class로 정의하는 것. `class RegisterRequest(BaseModel):` + `score: int`
= "score 칸은 정수여야 한다". 타입이 안 맞으면 내 함수가 실행되기 전에 422로 자동 거절.
값은 점 표기로 꺼냄: `register.score` (딕셔너리의 `["score"]`와 다름).

## AI

### 임베딩 (embedding)
텍스트를 의미를 담은 숫자 벡터로 바꾸는 것. `"연차" → [0.2, 0.8, ...]`
의미가 비슷한 문장은 벡터도 비슷해짐 → 숫자 비교로 "의미 검색"이 가능해짐.

### 코사인 유사도 (cosine similarity)
두 벡터가 얼마나 비슷한지 계산하는 방법. 1에 가까울수록 의미가 비슷.
"휴가 며칠?"과 "연차는 15일" 이 글자는 달라도 유사도가 높게 나옴.

### RAG (Retrieval-Augmented Generation)
AI가 학습한 것 외에 **내가 준 문서**에서도 답하게 만드는 기술.
흐름: 문서를 청크로 분리 → 임베딩 → 질문과 유사한 청크 검색 → 그것만 AI에 전달 → 답변.
토큰 절약 + 정확도 향상.

### 청크 (chunk)
문서를 검색 가능한 조각으로 자른 단위. 보통 문단(`\n\n`) 기준.
질문이 오면 전체 문서가 아니라 관련 청크만 AI한테 보냄.

### 벡터 DB (예: ChromaDB)
임베딩 벡터를 파일로 저장해두는 DB. 핵심: **임베딩 계산을 처음 한 번만** 하고,
그 다음부터는 저장된 벡터에서 바로 검색. 문서 10만 개여도 매번 재계산 안 함.

### Agent (에이전트)
질문을 받으면 AI가 **스스로 판단해서 도구를 선택·실행**하는 구조.
일반 AI: 질문 → 답변. Agent: 질문 → 도구 필요성 판단 → 도구 실행 → 결과로 답변.
도구는 `@tool` 붙인 함수. AI는 함수의 docstring을 읽고 언제 쓸지 결정.

### System Prompt
AI에게 역할과 규칙을 정해주는 숨은 지시문. `{"role": "system", "content": "한국어로만 답해"}`
사용자는 못 보지만 모든 대화에 적용됨. 개발자가 최적 프롬프트를 미리 설계해두는 자리.

### 토큰 (token)
AI가 텍스트를 세는 단위 (단어 조각). API 요금과 입력 한도가 토큰 기준.
RAG가 관련 청크만 보내는 이유 = 토큰 절약.

## 데이터 / 파이썬

### CSV
쉼표로 구분된 텍스트 파일. 첫 줄 = 열 이름(헤더).
**함정: 쉼표 뒤 공백도 데이터의 일부.** `name, score` → 열 이름이 `" score"`가 됨 → KeyError.

### 불린 필터링 (pandas)
`df[df["score"] >= 80]` — 두 단계로 읽는다:
①안쪽이 행마다 `[True, False, ...]` 목록을 **만들고** ②바깥 `df[...]`가 True인 행만 남김.

### 누적 vs 교체
변수를 읽을 때 첫 질문: "이 변수는 쌓이는 놈인가, 갈리는 놈인가."
- 누적: `total += n` (기존 값에 더함) — 합계 낼 때
- 교체: `best = n` (기존 값을 버림) — 최고/최저값 찾을 때
`+=` 안에 이미 덧셈이 있다. `total += total + n`은 두 배 뻥튀기 버그.

### 초기값의 가정
`best = 0`으로 시작하면 "모든 값이 0 이상"이라는 숨은 가정에 기댐.
안전한 방법: **첫 번째 실제 데이터**로 시작 (`best = data[0]["점수"]`).

### shadowing (이름 가리기)
`sum = 0`처럼 파이썬 내장 함수 이름을 변수로 쓰면 함수가 덮여서 못 쓰게 됨.
`sum`, `list`, `max`, `min`, `len` 등은 변수명으로 쓰지 말 것.

### f-string
`f"평균: {value}"` — 문자열 안 `{}`에 값이 끼워짐.
**함정: `{}`가 구멍이 되는 건 f-string 안에서만.** 문자열 밖에서 `{값}`은 집합(set)이 됨.
딕셔너리 값 안에서 꺼낼 땐 따옴표 종류를 다르게: `f"{student['name']}"`

### 경로 기준
`pd.read_csv("...")` 같은 상대 경로는 **파일 위치가 아니라 터미널이 서 있는 위치** 기준.
`FileNotFoundError`가 나면 터미널 위치부터 확인.

### .env / API 키 관리
API 키를 코드에 직접 쓰면 GitHub 푸시 시 노출·차단됨.
`.env` 파일에 키 저장 → `load_dotenv()` + `os.getenv("키이름")`으로 불러옴 → `.env`는 `.gitignore`에 등록.
